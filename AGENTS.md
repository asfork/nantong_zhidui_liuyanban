# AGENTS.md

## 项目目标

本仓库用于开发一个与老内网网站代码和 PHP 配置相互独立的留言板系统。

- 老网站只增加一个跳转链接，不在本仓库修改老项目代码。
- 留言板部署在同一台 Windows 服务器上的独立 Apache/PHP 运行环境。
- 正式访问地址固定为：`http://IP:新端口/liuyanban/`。
- 留言板复用老数据库中的管理员账号数据，但不得修改管理员账号、密码或表结构。
- Docker 仅用于开发和验证；生产交付必须支持传统 Apache/PHP 部署。

## 固定运行环境

开发、测试和交付均以以下版本为兼容基线，不得擅自升级：

- 生产操作系统：Windows 10 x64
- 生产集成环境：优先使用 phpStudy
- Apache：2.4.39（必须精确匹配）
- PHP：7.3.4
- MySQL：5.7.26
- 时区：`Asia/Shanghai`
- 文件编码：UTF-8，无 BOM
- 数据库字符集：`utf8mb4`
- 数据库排序规则：`utf8mb4_unicode_ci`

Docker 中必须验证并记录实际版本，不能只依赖镜像标签：

```sh
httpd -v
php -v
mysql --version
```

如果所选基础镜像无法提供 Apache 2.4.39，应构建并固定包含该版本的镜像，不得静默使用其他 Apache 版本。

## 架构边界

- 新留言板拥有独立源码目录、Apache 实例、PHP 配置、日志和 Session。
- 新 Apache 使用独立端口，不占用老网站端口。
- `/liuyanban/` 是应用的固定基础路径；所有链接、表单、跳转和静态资源路径都必须支持此前缀，禁止假定应用部署在 `/`。
- 推荐将 `/liuyanban/` 映射到项目的 `public/` 目录，非公开代码和配置不得位于可直接访问的 Web 根目录内。
- 不依赖老项目框架、路由、Session、Cookie 或公共 PHP 文件。
- 老网站与留言板之间只有普通超链接和管理员数据读取关系。
- 不修改老项目的 Apache/PHP 配置；留言板所需配置放入新实例。如果部署确实需要修改老实例，必须先说明原因并获得用户确认。

## 技术与兼容性要求

- 所有后端代码必须兼容 PHP 7.3.4。
- 禁止使用 PHP 7.4 及以上语法或 API，例如箭头函数、类型化属性和空合并赋值运算符。
- 数据访问统一使用 PDO MySQL 和预处理语句。
- SQL 必须兼容 MySQL 5.7.26，禁止使用 MySQL 8 专属语法或特性。
- 优先使用少依赖、可离线部署的实现。新增运行时依赖前必须确认其支持 PHP 7.3、可随部署包交付且不依赖生产服务器联网安装。
- 不假定存在老项目的 PHP 扩展。基础实现只可依赖常见扩展：`pdo_mysql`、`mysqli`、`mbstring`、`session`、`json`、`openssl`、`curl`、`fileinfo`、`gd`、`zip`。
- 若某功能必须使用额外扩展，必须提供启动前检查和明确错误提示，并更新部署文档。
- 前端优先支持 Chromium 系列浏览器，并尽量兼容较低版本。避免依赖可选链、空值合并、原生 ES Modules 等较新语法；需要使用时必须经过转译并随部署包交付产物。
- 页面优先采用服务端渲染和渐进增强，不要求 IE 兼容。

## 数据库约定

开发环境初始化一个空的 `zhidui_nantong` 数据库，并在其中创建测试 `admin` 表和留言板独立表。留言板表统一使用 `liuyan_` 前缀，避免与老项目表冲突。

最低表集合：

- `admin`：仅供开发验证的管理员账号表。
- `liuyan_message`：留言、审核状态、展示状态、来源 IP、创建和更新时间。
- `liuyan_reply`：管理员回复及回复时间。
- `liuyan_operation_log`：管理员登录及审核、回复、隐藏、恢复、删除等操作日志。

数据库设计规则：

- 使用 InnoDB、`utf8mb4`、`utf8mb4_unicode_ci`。
- 主键优先使用无符号整数自增字段。
- 时间字段含义必须明确，应用和数据库统一使用 `Asia/Shanghai`。
- 留言正文等用户输入必须设置合理长度上限，不得使用无界输入。
- 删除默认采用可审计的软删除；除非用户明确要求，不执行不可恢复的物理删除。
- 所有结构变更通过可重复执行或版本化的 SQL 迁移文件交付，不得只手工修改开发数据库。

生产环境优先使用独立数据库账号，并按最小权限授权：

- 对实际 `admin` 表只授予 `SELECT`。
- 对 `liuyan_*` 表授予业务所需的 `SELECT`、`INSERT`、`UPDATE`、`DELETE`。
- 应用不得更新或删除 `admin` 表数据。

数据库名、表名、字段映射和连接参数必须配置化，不得散落硬编码在业务代码中。

## 管理员认证

开发用 `admin` 表暂定字段如下：

- `id`：管理员 ID
- `username`：唯一用户名
- `password`：密码摘要，长度至少 255
- `user_type`：管理员类型

开发测试密码默认使用 PHP 原生 bcrypt：

```php
password_hash($password, PASSWORD_BCRYPT);
password_verify($password, $hash);
```

老系统真实字段名和密码算法暂时未知，因此认证层必须隔离并配置化，至少保留以下配置入口：

- 管理员表名
- ID、用户名、密码和用户类型字段名
- 允许登录后台的用户类型列表
- 密码校验驱动

默认密码驱动为 `bcrypt`。部署时允许替换为经确认的老系统算法（如 `md5`、加盐 MD5 或自定义校验器），但不得为了兼容而自动降级或尝试多种算法。禁止记录明文密码和完整密码摘要。

留言板后台使用独立 Session：

- Session Cookie 名称使用 `LIUYANBANSESSID` 或等价的项目专用名称。
- Cookie Path 设置为 `/liuyanban/`。
- Cookie 至少启用 `HttpOnly` 和 `SameSite=Lax`。
- 当前站点没有 SSL，不能依赖 `Secure` Cookie；代码和配置应保留启用 HTTPS 后设置 `Secure` 的入口。

## 基础业务范围

首个版本按以下范围实现，不额外扩展：

### 公开留言板

- 用户无需登录，匿名提交留言。
- 不采集或展示真实姓名、电话等身份信息，除非后续需求明确增加。
- 不支持附件上传。
- 提交成功后的留言默认进入“待审核”状态。
- 只有审核通过且未隐藏、未删除的留言可以在公开页面展示。
- 已公开留言可以展示管理员回复。
- 提交时记录来源 IP，但公开页面不得展示完整 IP。

### 管理后台

- 管理员使用复用的 `admin` 账号独立登录。
- 支持按状态查看、分页和检索留言。
- 支持审核通过或驳回。
- 支持回复留言。
- 支持隐藏和恢复留言。
- 支持软删除留言。
- 所有改变留言状态或内容的后台操作必须写入操作日志。

### 建议状态模型

- 审核状态：`pending`、`approved`、`rejected`
- 展示状态：`visible`、`hidden`
- 删除标记：通过 `deleted_at` 表示软删除

不得用一个含义不清的数字字段同时表达审核、隐藏和删除状态。

## 安全要求

- 所有 SQL 使用参数化查询，禁止拼接用户输入。
- 所有写操作必须进行 CSRF 校验。
- 所有输出按 HTML 上下文转义，留言正文不得直接输出未过滤 HTML。
- 管理登录必须使用通用错误提示，避免泄露账号是否存在。
- 登录和匿名提交应具备基础频率限制；实现方式必须适合单机、内网和 PHP 7.3 环境。
- 校验请求方法、Content-Type、字段长度和字符编码。
- 获取客户端 IP 时默认使用服务器直接连接地址；只有明确配置可信代理后，才可读取 `X-Forwarded-For` 等代理头。
- 应用日志不得记录密码、Session ID、数据库密码或完整敏感请求数据。
- 即使运行在纯内网，也必须保留上述安全措施。

## 配置管理

- 数据库连接、基础路径、Session、密码驱动、管理员字段映射和日志路径均通过环境变量或站点外部配置提供。
- 仓库提供无密钥的配置示例，但不得提交真实账号、密码、IP 或生产配置。
- 开发默认值只能用于本地 Docker，生产启动时缺少关键配置应立即失败并给出明确提示。
- 基础路径默认值为 `/liuyanban`，新端口不得写死在业务代码中。
- 日志、缓存和 Session 目录必须可配置并在启动检查中验证可写性。

## 推荐目录职责

实现时优先保持以下边界；若采用不同结构，应维持同等的公开/私有隔离：

```text
app/                 PHP 应用代码
config/              无密钥默认配置和配置装配
database/migrations/ 数据库迁移 SQL
docker/              开发环境镜像和 Apache/PHP 配置
public/              唯一允许对外暴露的目录
tests/               自动化测试
var/                 日志、缓存和 Session，禁止提交运行数据
```

## 测试与验收

每次变更应按风险执行相应验证，至少覆盖：

- PHP 7.3语法检查。
- Docker 中 Apache 2.4.39、PHP 7.3.4、MySQL 5.7.26实际版本检查。
- `/liuyanban/` 基础路径下页面、静态资源、表单和跳转均可工作。
- 匿名提交后状态为待审核。
- 未审核、驳回、隐藏和删除留言不会公开展示。
- 审核通过后可展示，管理员回复可正确关联。
- 后台未登录不可访问，登录、退出和 Session 隔离有效。
- 管理操作均产生操作日志。
- SQL 注入、存储型 XSS 和 CSRF 基础测试。
- 中文、常见标点和四字节 UTF-8字符能够正确保存和展示。
- Windows 10 x64 + phpStudy 环境完成最终部署冒烟测试。

如果仓库添加了统一测试或检查命令，应同步更新本文件和 README，确保新开发者能通过一组明确命令完成验证。

